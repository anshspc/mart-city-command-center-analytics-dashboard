# Smart City Command Center Analytics Dashboard

🌐 **Live Dashboard Link**: [https://mart-city-command-center-analytics.onrender.com](https://mart-city-command-center-analytics.onrender.com)

Welcome to the **Smart City Command Center Analytics Dashboard** project. This is a full-stack, portfolio-grade data engineering and predictive analytics application designed to monitor, analyze, and forecast municipal operations using data from multiple civic departments.

---

## 🛠️ Technology Stack
* **Data Processing & Engineering**: Python (Pandas, NumPy, SQLite, Openpyxl)
* **Machine Learning & Forecasting**: Scikit-learn, Joblib (Random Forests, Auto-regressive Forecasting)
* **Backend Services**: FastAPI, Uvicorn, Pydantic
* **Frontend Web Dashboard**: Custom HTML5, Vanilla CSS3 (Glassmorphism design), Vanilla JS, Chart.js, Leaflet.js Maps
* **Secondary Python Dashboard**: Streamlit, Plotly Express
* **Automated Reporting Engine**: ReportLab (PDF generation), Openpyxl (Multi-tab Excel styling)
* **Data Visualization Integration**: Power BI Desktop (includes custom DAX measures & schema guide)

## 🚀 Setup & Execution Guide

Follow these steps to set up the virtual environment, generate the datasets, ingest the database, train the machine learning models, and launch the dashboard interfaces:

### Step 1: Initialize Virtual Environment
1. Navigate to the project root directory:
   ```bash
   cd smart-city-command-center
   ```
2. Create a Python virtual environment:
   ```bash
   python3 -m venv .venv
   ```
3. Activate the environment:
   * **macOS / Linux**: `source .venv/bin/activate`
   * **Windows (Cmd)**: `.venv\Scripts\activate`
4. Install all dependencies:
   ```bash
   pip install -r requirements.txt
   ```

### Step 2: Run the Data Engineering & ML Pipeline
1. **Generate Raw Datasets** (creates over 17,000 combined rows of temporal/spatial data):
   ```bash
   python etl/generate_sample_data.py
   ```
2. **Execute ETL Database Ingestion** (cleanses raw logs and loads them into a normalized SQLite database with configured query indexes):
   ```bash
   python etl/pipeline.py
   ```
3. **Train Machine Learning Models** (trains, evaluates, and saves the 4 predictive models):
   ```bash
   python models/train.py
   ```
4. **Pre-generate Operational Reports** (creates the PDF & styled Excel files in the `reports/` folder):
   ```bash
   python reports/generator.py
   ```

### Step 3: Launch the Frontends

#### Option A: Custom Glassmorphism Web App (Recommended Showcase)
1. Start the FastAPI backend server:
   ```bash
   uvicorn backend.main:app --reload
   ```
2. Open your web browser and navigate to:
   ```
   http://127.0.0.1:8000/
   ```
   * **Features**: Live KPI updates, Chart.js trend visualizations, interactive Leaflet.js complaint marker mapping, data pipeline sync utility, and active forms for ML inference.

#### Option B: Python-Native Streamlit Dashboard
1. Start the Streamlit server:
   ```bash
   streamlit run dashboard/app.py
   ```
2. Open your web browser (usually opens automatically) at:
   ```
   http://localhost:8501
   ```
   * **Features**: Plotly interactive charts, scatter maps, file download hub, and model forecast sandbox sliders.

## 📊 Analytics & Predictions Breakdown

1. **Executive Command Summary**: High-level view of complaints, resolution ratios, outage metrics, and sanitation logistics.
2. **Traffic Congestion Index**: Displays peaks cycles, sensor speed readings, weather impacts, and includes an ML regression playground.
3. **Utilities Grid**: Monitors water consumption, leaks, load factor variables, power blackout statistics, and waste management sectors.
4. **Citizen Request Map**: Interactive geospatial view of tickets color-coded by lifecycle state (Open = Red, In Progress = Orange, Resolved = Green).
5. **Report Exports**: Download automated PDF reports and Excel sheets showing current operational metrics.
